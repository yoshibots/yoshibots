SELinux¶

Install¶

Package¶

semange command¶

dnf install -y policycoreutils-python-utils # CentOS8,RHEL9,
yum install -y policycoreutils-python       # CentOS7,

基本形¶

semanage port --list #
semanage -a -t <LABELTYPE> -p tcp <PORT_NUMBER> #

Warning

無効状態から直接有効にするとラベルが適切でないものがある場合起動しなくなってしまう場合がある。その為、下記の手順を実施して一度Permissiveで起動後に有効化する。

# ルートディレクトリに.autorelabel を作成する
root> touch .autorelabel

# Permissiveに変更して起動させる

# Enforcingで起動 > vi /etc/selinux/config ——————————

# 参照権限の付与 > semanage fcontext -a -t httpd_sys_content_t “<XXX>”

> setsebool -P httpd_can_network_connect_db 1 # 要確認 -> webserver経由でDBアクセスできる設定 > setsebool -P httpd_can_network_connect=on

## よく使うコマンド————————————

semanage port -a -t <ssh_port_t> -p tcp <PORT_NUMBER> # Ex.ssh_port_t  # portの追加
semanage port -d -t <TYPE_LABEL> -p tcp <PORT_NUMBER> # portの削除
semanage fcontext -l  # コンテキスト定義ルールの一覧表示
semanage fcontext --list | grep httpd_config_t  # 絞り込み

seinfo -t  # SELinux のタイプ(type)一覧表示
seinfo -x -c  # SELinuxのClassとPermission一覧表示

semanage fcontext -d -e </var/www> </var/test_www>  # 定義したRuleの削除

getsebool -a  # grep で絞り込む getsebool -a | grep http  # 組み込みルールのステータスを確認

# 設定状態の確認(SELinux の個別追加設定した内容を表示(ただしsemodule等で追加したモジュール情報は見れない))
 semanage export

 sesearch -A -s httpd_t -b httpd_can_network_connect_db -p name_connect  # AVCの確認

# 上記で許可されているPortの確認 > seinfo –protocol=tcp –port | grep -E ‘mongod_port_t|gds_db_port_t|mssql_port_t|oracle_port_t|postgresql_port_t|mysqld_port_t’

Example¶

sshの…¶

semanage port --list | grep ssh  # sshのLabel確認
semanage port -a -t ssh_port_t -p tcp 20000 # 20000ポートにssh_port_tのラベルを設定

http¶

semanage port --list | grep http  # httpのLabel確認
semanage port -a -t http_port_t -p tcp 20000 # 20000ポートにhttp_port_tのラベルを設定
